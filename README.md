## ❌ 没有它，你的 AI 是这样的

- 你说「做个用户管理页面」，AI 立刻开始写代码 —— 没人问过**给谁用、什么场景、核心指标是什么**
- UI 做出来像「默认模板拼凑」，没有主次、没有重心、没有设计感
- 需求写一半就开始动手，做到一半发现方向错了，全部推倒重来
- AI 永远说「好的，马上做」—— **你需要的不是舔狗，是诤友**
- 每次需求都要填一堆表，问完十几个问题才开始动 —— **决策者精力全耗在回答上**

## ✅ 有了它，你的 AI 是这样的

- 只问 **3 件关键事**，其余由 AI 自主提案——决策者的工作是「审阅 + 拍板」，不是「填表 + 回答」
- 给出**七段式 PRD**，清晰标注哪段由人确认、哪段由 AI 提案，防止互相推责
- PRD 完成后 **自动执行 Pre-mortem** 风险扫描（Tiger / Elephant / Paper Tiger），不需要额外触发
- **审美把关**：7 条验收标准逐条打分，任意一条不通过直接打回整改
- 发现问题**拒绝执行**，附上理由和更好的方案，要求优先级排序必须给出 ICE / RICE / Opportunity Score

### 方式一：一键安装（推荐）

```bash
# Mac / Linux
curl -fsSL https://raw.githubusercontent.com/hew797/ai-pm-prompt/main/install.sh | bash -s -- /你的项目路径

# 例如
curl -fsSL https://raw.githubusercontent.com/hew797/ai-pm-prompt/main/install.sh | bash -s -- /Users/me/my-project
```

### 方式二：手动安装

```bash
# 克隆到本地
git clone https://github.com/hew797/ai-pm-prompt.git

# 复制核心文件到你的项目
cp ai-pm-prompt/AGENTS.md /你的项目路径/
```

### 方式三：直接复制粘贴

打开 [AGENTS.md](./AGENTS.md)，复制全部内容，粘贴到你的 AI 工具的系统提示词中。

## 🔧 适配你的 AI 工具

安装完成后，根据你使用的工具，按以下方式激活：

### Gemini CLI

```bash
# AGENTS.md 会被自动放入项目根目录
# Gemini CLI 启动时自动读取 AGENTS.md 作为系统指令
# 无需额外配置，直接开始对话即可
gemini
```

> Gemini CLI 会自动读取项目根目录的 `AGENTS.md`。如果你的项目已有 `AGENTS.md`，安装脚本会备份原文件。

### Cursor

1. 打开 Cursor 设置 → `Rules for AI`
2. 添加以下内容：

```
请阅读项目根目录的 AGENTS.md 文件，严格按照其中定义的角色和流程工作。
```

> 或者将 AGENTS.md 的内容直接粘贴到 Cursor 的 Project Rules（`.cursor/rules`）中。

### Windsurf

1. 在项目根目录创建 `.windsurfrules` 文件
2. 写入：

```
请阅读项目根目录的 AGENTS.md 文件，严格按照其中定义的角色和流程工作。
```

### Claude Code

```bash
# Claude Code 支持 CLAUDE.md 作为项目指令
echo "请阅读并严格执行 AGENTS.md 中定义的角色和流程。" > CLAUDE.md
```

## 🎯 核心能力展示

### 能力 1：关键三问 + AI 自主提案（v2.0 新交互）

你说：
> 「做一个用户管理页面」

AI 产品经理**不会直接写代码**，先问这 3 件事：

```
我需要确认 3 件事（回答后立刻给方案）：

① 目标：这个功能帮「谁」在「什么场景」做「什么事」？
② 约束：有什么不能做的？（时间节点 / 技术限制 / 业务红线）
③ 成功：用户完成什么动作，你才觉得这个功能做成了？
```

你回答完毕，AI **立刻输出功能卡草稿**，你只需要说「对」「改成这样：___」「去掉这个」：

```
## 功能卡（AI 提案版）

一句话目标：帮助系统管理员在运营后台快速查找、编辑和停用用户账户
用户角色：系统管理员                      ← AI 推断
触发场景：日常运营维护 / 处理用户投诉      ← AI 推断
首屏焦点：搜索框 + 用户列表               ← AI 推断
信息优先级：
  A（核心）：用户名、账号状态、最后登录    ← AI 排列
  B（辅助）：注册时间、角色权限            ← AI 排列
  C（低频）：操作日志、登录记录            ← AI 排列
功能范围 In：搜索、筛选、查看、编辑、停用  ← AI 提议
功能范围 Out：批量导入、权限配置           ← AI 提议

⚠️ 以上标注「AI 推断」的内容请审阅后确认。
```

### 能力 2：PRD 七段式（职责清晰，防止互相推责）

每段标注「谁负责」，决策者只需确认 3 个决策点：

| 段落 | 负责方 |
|------|--------|
| 1. 背景与北极星目标 | **决策者确认**（来自关键三问①） |
| 2. 用户角色 + 价值主张 | **AI 提案 → 决策者审阅** |
| 3. 系统蓝图 | **AI 提案 → 决策者审阅** |
| 4. 核心业务规则与算法 | **决策者确认**（红线只有人能定） |
| 5. 功能范围 In / Out | **AI 提案 → 决策者审阅** |
| 6. 验收标准 | **决策者确认**（来自关键三问③） |
| 7. 开放问题 | **AI 自动生成** |

PRD 完成后，AI 自动输出 Pre-mortem：

```
🐯 Tiger（真实风险，必须行动）：……
🐘 Elephant（没人说出口的隐患）：……
👻 Paper Tiger（过度担心的风险）：……
```

### 能力 3：UI 审美验收（不是「能用就行」，是「被设计过」）

AI 产品经理会按 7 条标准逐项打分：

```
1. ✅ 任务清晰：用户 3 秒内知道这页干什么
2. ❌ 首屏重心：搜索框被挤到角落，应该成为视觉中心
3. ❌ 信息优先级：所有列都一样宽，重要信息没有突出
4. ✅ 视觉层级：块面和留白基本合理
5. ❌ 审美完成度：纯白底 + 灰边 + 蓝按钮，像「默认组件堆叠」
6. ✅ 文案可懂度：标题和操作按钮文案清晰
7. ❌ 异常状态：空搜索结果没有引导文案

→ 4/7 不通过，打回整改，附具体修改建议
```

### 能力 4：诤友人格（不做舔狗）

```
你：帮我加一个弹窗，用户每次进来都要先看公告

AI 产品经理：
[反问] 这个弹窗每次都弹，用户第 100 次进来还要看吗？
建议：首次显示用公告条，非首次用角标提示，
点击才展开。理由：强制弹窗是用户流失的第一杀手。
你坚持要弹窗我也做，但风险已告知。
```

## ⚡ 快捷指令

| 指令 | 用途 |
|------|------|
| `/pm-spec [功能意图]` | 关键三问 + AI 自主功能卡（决策者审阅确认即可） |
| `/pm-prd [功能意图]` | 直接生成完整 PRD 草稿 + Pre-mortem，决策者只需确认 3 个决策点 |
| `/pm-layout [模块名]` | 页面区块划分 + 首屏重心设计 |
| `/pm-critique [模块名]` | 7 条标准逐项验收打分 |
| `/pm-audit [模块名]` | 3 个观感问题 + 3 条改动建议 |

## 💡 设计理念

1. **决策者只做 3 件事** — 目标、约束、成功标准。其余由 AI 提案，决策者审阅
2. **AI 不是助手，是技术合伙人** — 它应该敢于说不，不允许只提问题不给方案
3. **审美不是加分项，是及格线** — 后台系统也值得被认真设计
4. **问题要有数据支撑** — 优先级排序必须给框架（ICE / RICE / Opportunity Score），禁止只写「高优先级」
5. **克制不等于苍白** — 允许高级感，禁止廉价感
