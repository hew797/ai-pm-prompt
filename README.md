## ❌ 没有它，你的 AI 是这样的

- 你说"做个用户管理页面"，AI 立刻开始写代码 —— 没人问过**给谁用、什么场景、核心指标是什么**
- UI 做出来像"默认模板拼凑"，没有主次、没有重心、没有设计感
- 需求写一半就开始动手，做到一半发现方向错了，全部推倒重来
- AI 永远说"好的，马上做" —— **你需要的不是舔狗，是诤友**

## ✅ 有了它，你的 AI 是这样的

- 收到需求先**追问三遍**：目标用户是谁？成功标准是什么？首屏重心在哪里？
- 给出**七段式 PRD**：背景目标 → 用户故事 → 系统蓝图 → 业务规则 → 功能范围 → 验收标准 → 开放问题
- **审美把关**：7 条验收标准逐条打分（任务清晰？首屏重心？信息优先级？视觉层级？审美完成度？文案可懂？异常状态？）
- 发现问题**拒绝执行**，附上理由和更好的方案

## 🚀 30 秒上手

### 方式一：一键安装（推荐）

```bash
# Mac / Linux
curl -fsSL https://raw.githubusercontent.com/hew797/ai-pm-prompt/main/install.sh | bash -s -- /你的项目路径

# 例如
curl -fsSL https://raw.githubusercontent.com/hew797/ai-pm-prompt/main/install.sh | bash -s -- /Users/me/my-project
```

### 方式二：手动安装

```bash
# 克隆到本地
git clone https://github.com/hew797/ai-pm-prompt.git

# 复制核心文件到你的项目
cp ai-pm-prompt/AGENTS.md /你的项目路径/
cp -r ai-pm-prompt/roles/ /你的项目路径/.ai-pm/
cp -r ai-pm-prompt/templates/ /你的项目路径/.ai-pm/
```

### 方式三：直接复制粘贴

打开 [AGENTS.md](./AGENTS.md)，复制全部内容，粘贴到你的 AI 工具的系统提示词中。

## 🔧 适配你的 AI 工具

安装完成后，根据你使用的工具，按以下方式激活：

### Gemini CLI

```bash
# AGENTS.md 会被自动放入项目根目录
# Gemini CLI 启动时自动读取 AGENTS.md 作为系统指令
# 无需额外配置，直接开始对话即可
gemini
```

> Gemini CLI 会自动读取项目根目录的 `AGENTS.md`。如果你的项目已有 `AGENTS.md`，安装脚本会备份原文件。

### Cursor

1. 打开 Cursor 设置 → `Rules for AI`
2. 添加以下内容：

```
请阅读项目根目录的 AGENTS.md 文件，严格按照其中定义的角色和流程工作。
同时阅读 .ai-pm/roles/pm-ux.md 作为产品经理技能库。
```

> 或者将 AGENTS.md 的内容直接粘贴到 Cursor 的 Project Rules（`.cursor/rules`）中。

### Windsurf

1. 在项目根目录创建 `.windsurfrules` 文件
2. 写入：

```
请阅读项目根目录的 AGENTS.md 文件，严格按照其中定义的角色和流程工作。
同时阅读 .ai-pm/roles/pm-ux.md 作为产品经理技能库。
```

### Claude Code

```bash
# Claude Code 支持 CLAUDE.md 作为项目指令
# 安装脚本会自动创建 CLAUDE.md 引用 AGENTS.md
# 或者手动添加：
echo "请阅读并严格执行 AGENTS.md 中定义的角色和流程。" > CLAUDE.md
```

## 🎯 核心能力展示

### 能力 1：需求蒸馏（把模糊想法变成清晰方案）

你说：
> "做一个用户管理页面"

AI 产品经理会输出：

```
一句话目标：帮助系统管理员在「运营后台」快速查找、编辑和停用用户账户
成功标志：管理员能在 10 秒内找到目标用户并完成操作
首屏焦点：用户搜索框 + 筛选结果列表
信息优先级：
  A（核心）：用户名、状态、最后登录时间
  B（辅助）：注册时间、角色
  C（低频）：登录日志、操作历史
页面基调：高效的管理工作台
```

### 能力 2：UI 审美验收（不是"能用就行"，是"被设计过"）

AI 产品经理会按 7 条标准逐项打分：

```
1. ✅ 任务清晰：用户 3 秒内知道这页干什么
2. ❌ 首屏重心：搜索框被挤到角落，应该成为视觉中心
3. ❌ 信息优先级：所有列都一样宽，重要信息没有突出
4. ✅ 视觉层级：块面和留白基本合理
5. ❌ 审美完成度：纯白底 + 灰边 + 蓝按钮，像"默认组件堆叠"
6. ✅ 文案可懂度：标题和操作按钮文案清晰
7. ❌ 异常状态：空搜索结果没有引导文案

→ 4/7 不通过，打回整改，附具体修改建议
```

### 能力 3：诤友人格（不做舔狗）

```
你：帮我加一个弹窗，用户每次进来都要先看公告

AI 产品经理：
[反问] 这个弹窗每次都弹，用户第 100 次进来还要看吗？
建议：首次显示用公告条，非首次用角标提示，
点击才展开。理由：强制弹窗是用户流失的第一杀手。
你坚持要弹窗我也做，但风险已告知。
```

## 💡 设计理念

1. **先问任务，再写代码** — 没有需求分析的代码都是技术债
2. **AI 不是助手，是技术合伙人** — 它应该敢于说不
3. **审美不是加分项，是及格线** — 后台系统也值得被认真设计
4. **一致性不是单调** — 同类模式要一致，但不同页面应有不同重心
5. **克制不等于苍白** — 允许高级感，禁止廉价感
